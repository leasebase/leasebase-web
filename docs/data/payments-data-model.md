# Payments Data Model

All tables live in the `payments_service` PostgreSQL schema (Aurora Serverless v2).
Amounts are stored as integers in the **smallest currency unit** (cents for USD).
All IDs are UUIDs generated by the database (`gen_random_uuid()`).
All tables include `created_at` and `updated_at` timestamps with `DEFAULT NOW()`.

## Entity Relationship Overview

```
payment_account (1 per org)
  │
  ├── payment_obligation (N per lease)
  │     │
  │     └── payment_transaction (N per obligation)
  │           │
  │           ├── refund (N per transaction)
  │           │
  │           └── ledger_entry (N per transaction)
  │
  ├── payout (N per account)
  │
  ├── dispute (N per account)
  │
  └── webhook_event (N, global)

payment_method (N per tenant)

rent_schedule (N per lease)
```

## 1. payment_account

Stripe Connect account linked to an organization.

**Owning service:** payments-service
**Key relationships:** org_id → auth-service/org, stripe_account_id → Stripe

```sql
CREATE TABLE payment_account (
  id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id              UUID NOT NULL UNIQUE,
  stripe_account_id   TEXT NOT NULL UNIQUE,
  account_type        TEXT NOT NULL DEFAULT 'OWNER',
    -- OWNER, PROPERTY_MANAGER, VENDOR (future)
  status              TEXT NOT NULL DEFAULT 'ONBOARDING_INCOMPLETE',
    -- NOT_STARTED, ONBOARDING_INCOMPLETE, PENDING_VERIFICATION,
    -- ACTIVE, RESTRICTED, DISCONNECTED
  charges_enabled     BOOLEAN NOT NULL DEFAULT FALSE,
  payouts_enabled     BOOLEAN NOT NULL DEFAULT FALSE,
  capabilities        JSONB DEFAULT '{}',
  requirements        JSONB DEFAULT '{}',
  platform_fee_percent NUMERIC(5,4) DEFAULT 0.0100,
    -- e.g., 0.0100 = 1%
  platform_fee_fixed  INTEGER DEFAULT 0,
    -- fixed fee in cents, added to percentage
  payout_schedule     JSONB DEFAULT '{"interval": "daily", "delay_days": 2}',
  metadata            JSONB DEFAULT '{}',
  created_at          TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at          TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_payment_account_org ON payment_account(org_id);
CREATE INDEX idx_payment_account_stripe ON payment_account(stripe_account_id);
```

## 2. payment_method

Tenant's saved payment method (Stripe PM reference).

**Owning service:** payments-service
**Key relationships:** tenant_id → tenant-service, stripe_payment_method_id → Stripe

```sql
CREATE TABLE payment_method (
  id                        UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id                 UUID NOT NULL,
  stripe_payment_method_id  TEXT NOT NULL UNIQUE,
  stripe_customer_id        TEXT NOT NULL,
  type                      TEXT NOT NULL,
    -- us_bank_account, card
  last_four                 TEXT,
  bank_name                 TEXT,
  card_brand                TEXT,
  is_default                BOOLEAN NOT NULL DEFAULT FALSE,
  is_verified               BOOLEAN NOT NULL DEFAULT FALSE,
  status                    TEXT NOT NULL DEFAULT 'ACTIVE',
    -- ACTIVE, EXPIRED, DETACHED
  metadata                  JSONB DEFAULT '{}',
  created_at                TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at                TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_payment_method_tenant ON payment_method(tenant_id);
```

## 3. rent_schedule

Recurring rent schedule generated from a lease. Defines what is owed and when.

**Owning service:** payments-service (populated from lease-service data)
**Key relationships:** lease_id → lease-service, property_id → property-service

```sql
CREATE TABLE rent_schedule (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id          UUID NOT NULL,
  lease_id        UUID NOT NULL,
  property_id     UUID NOT NULL,
  unit_id         UUID NOT NULL,
  tenant_id       UUID NOT NULL,
  amount          INTEGER NOT NULL,
  currency        TEXT NOT NULL DEFAULT 'usd',
  frequency       TEXT NOT NULL DEFAULT 'MONTHLY',
    -- MONTHLY, WEEKLY, BIWEEKLY
  day_of_month    INTEGER,
    -- 1-28 for MONTHLY
  start_date      DATE NOT NULL,
  end_date        DATE,
  is_active       BOOLEAN NOT NULL DEFAULT TRUE,
  metadata        JSONB DEFAULT '{}',
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_rent_schedule_lease ON rent_schedule(lease_id);
CREATE INDEX idx_rent_schedule_org ON rent_schedule(org_id);
CREATE INDEX idx_rent_schedule_active ON rent_schedule(is_active) WHERE is_active = TRUE;
```

## 4. payment_obligation

A specific amount due on a specific date. Generated from rent_schedule or created manually (one-time fees).

**Owning service:** payments-service
**Key relationships:** lease_id → lease-service, rent_schedule_id → rent_schedule

```sql
CREATE TABLE payment_obligation (
  id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id            UUID NOT NULL,
  lease_id          UUID NOT NULL,
  tenant_id         UUID NOT NULL,
  rent_schedule_id  UUID REFERENCES rent_schedule(id),
  type              TEXT NOT NULL DEFAULT 'RENT',
    -- RENT, LATE_FEE, DEPOSIT, APPLICATION_FEE, OTHER
  amount            INTEGER NOT NULL,
  amount_paid       INTEGER NOT NULL DEFAULT 0,
  currency          TEXT NOT NULL DEFAULT 'usd',
  due_date          DATE NOT NULL,
  status            TEXT NOT NULL DEFAULT 'PENDING',
    -- PENDING, PARTIAL, PAID, PAST_DUE, CANCELED, WAIVED
  description       TEXT,
  metadata          JSONB DEFAULT '{}',
  created_at        TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at        TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_obligation_org ON payment_obligation(org_id);
CREATE INDEX idx_obligation_lease ON payment_obligation(lease_id);
CREATE INDEX idx_obligation_tenant ON payment_obligation(tenant_id);
CREATE INDEX idx_obligation_due ON payment_obligation(due_date, status);
CREATE INDEX idx_obligation_status ON payment_obligation(status) WHERE status IN ('PENDING', 'PARTIAL', 'PAST_DUE');
```

## 5. payment_transaction

A single payment attempt against an obligation. Maps 1:1 to a Stripe PaymentIntent.

**Owning service:** payments-service
**Key relationships:** obligation_id → payment_obligation, stripe_payment_intent_id → Stripe

```sql
CREATE TABLE payment_transaction (
  id                        UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id                    UUID NOT NULL,
  obligation_id             UUID NOT NULL REFERENCES payment_obligation(id),
  tenant_id                 UUID NOT NULL,
  stripe_payment_intent_id  TEXT UNIQUE,
  stripe_charge_id          TEXT,
  stripe_checkout_session_id TEXT,
  stripe_transfer_id        TEXT,
  stripe_balance_txn_id     TEXT,
  destination_account_id    UUID REFERENCES payment_account(id),
  amount                    INTEGER NOT NULL,
  application_fee_amount    INTEGER NOT NULL DEFAULT 0,
  currency                  TEXT NOT NULL DEFAULT 'usd',
  payment_method_type       TEXT,
    -- us_bank_account, card
  status                    TEXT NOT NULL DEFAULT 'INITIATED',
    -- INITIATED, PROCESSING, SUCCEEDED, FAILED, CANCELED, REFUNDED, PARTIALLY_REFUNDED
  failure_code              TEXT,
  failure_message           TEXT,
  refunded_amount           INTEGER NOT NULL DEFAULT 0,
  metadata                  JSONB DEFAULT '{}',
  processed_at              TIMESTAMPTZ,
  created_at                TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at                TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_txn_org ON payment_transaction(org_id);
CREATE INDEX idx_txn_obligation ON payment_transaction(obligation_id);
CREATE INDEX idx_txn_tenant ON payment_transaction(tenant_id);
CREATE INDEX idx_txn_stripe_pi ON payment_transaction(stripe_payment_intent_id);
CREATE INDEX idx_txn_status ON payment_transaction(status);
```

## 6. payout

Tracks Stripe payouts to connected accounts.

**Owning service:** payments-service
**Key relationships:** payment_account_id → payment_account, stripe_payout_id → Stripe

```sql
CREATE TABLE payout (
  id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  payment_account_id  UUID NOT NULL REFERENCES payment_account(id),
  org_id              UUID NOT NULL,
  stripe_payout_id    TEXT NOT NULL UNIQUE,
  amount              INTEGER NOT NULL,
  currency            TEXT NOT NULL DEFAULT 'usd',
  status              TEXT NOT NULL DEFAULT 'PENDING',
    -- PENDING, IN_TRANSIT, PAID, FAILED, CANCELED
  arrival_date        DATE,
  method              TEXT DEFAULT 'standard',
  failure_code        TEXT,
  failure_message     TEXT,
  metadata            JSONB DEFAULT '{}',
  created_at          TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at          TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_payout_account ON payout(payment_account_id);
CREATE INDEX idx_payout_org ON payout(org_id);
CREATE INDEX idx_payout_stripe ON payout(stripe_payout_id);
```

## 7. refund

Refund issued against a payment transaction.

**Owning service:** payments-service
**Key relationships:** payment_transaction_id → payment_transaction, stripe_refund_id → Stripe

```sql
CREATE TABLE refund (
  id                      UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  payment_transaction_id  UUID NOT NULL REFERENCES payment_transaction(id),
  org_id                  UUID NOT NULL,
  stripe_refund_id        TEXT UNIQUE,
  amount                  INTEGER NOT NULL,
  currency                TEXT NOT NULL DEFAULT 'usd',
  status                  TEXT NOT NULL DEFAULT 'PENDING',
    -- PENDING, SUCCEEDED, FAILED, CANCELED
  reason                  TEXT,
    -- tenant_request, duplicate, fraudulent, other
  refund_application_fee  BOOLEAN NOT NULL DEFAULT FALSE,
  failure_reason          TEXT,
  metadata                JSONB DEFAULT '{}',
  created_at              TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at              TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_refund_txn ON refund(payment_transaction_id);
CREATE INDEX idx_refund_org ON refund(org_id);
```

## 8. dispute

Stripe dispute (chargeback) on a payment.

**Owning service:** payments-service
**Key relationships:** payment_transaction_id → payment_transaction, stripe_dispute_id → Stripe

```sql
CREATE TABLE dispute (
  id                      UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  payment_transaction_id  UUID NOT NULL REFERENCES payment_transaction(id),
  org_id                  UUID NOT NULL,
  stripe_dispute_id       TEXT NOT NULL UNIQUE,
  amount                  INTEGER NOT NULL,
  currency                TEXT NOT NULL DEFAULT 'usd',
  status                  TEXT NOT NULL DEFAULT 'OPEN',
    -- OPEN, WON, LOST, CLOSED
  reason                  TEXT,
    -- duplicate, fraudulent, subscription_canceled, product_unacceptable, etc.
  evidence_due_by         TIMESTAMPTZ,
  metadata                JSONB DEFAULT '{}',
  created_at              TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at              TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_dispute_txn ON dispute(payment_transaction_id);
CREATE INDEX idx_dispute_org ON dispute(org_id);
CREATE INDEX idx_dispute_status ON dispute(status) WHERE status = 'OPEN';
```

## 9. ledger_entry

Append-only financial ledger. Every money movement is recorded here.

**Owning service:** payments-service
**Key relationships:** references multiple entities; stripe_object_id → Stripe

```sql
CREATE TABLE ledger_entry (
  id                      UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id                  UUID NOT NULL,
  lease_id                UUID,
  tenant_id               UUID,
  payment_transaction_id  UUID REFERENCES payment_transaction(id),
  obligation_id           UUID REFERENCES payment_obligation(id),
  refund_id               UUID REFERENCES refund(id),
  payout_id               UUID REFERENCES payout(id),
  dispute_id              UUID REFERENCES dispute(id),
  type                    TEXT NOT NULL,
    -- CHARGE, PAYMENT, PLATFORM_FEE, REFUND, PAYOUT,
    -- DISPUTE, DISPUTE_REVERSAL, CREDIT, LATE_FEE,
    -- PAYMENT_REVERSAL, VENDOR_PAYOUT
  amount                  INTEGER NOT NULL,
    -- positive = credit/inflow, negative = debit/outflow
  currency                TEXT NOT NULL DEFAULT 'usd',
  description             TEXT,
  stripe_object_id        TEXT,
    -- pi_xxx, re_xxx, po_xxx, etc.
  stripe_balance_txn_id   TEXT,
  status                  TEXT NOT NULL DEFAULT 'POSTED',
    -- PENDING, POSTED, REVERSED
  effective_date          DATE NOT NULL DEFAULT CURRENT_DATE,
  metadata                JSONB DEFAULT '{}',
  created_at              TIMESTAMPTZ NOT NULL DEFAULT NOW()
  -- NOTE: no updated_at — ledger entries are append-only
);

CREATE INDEX idx_ledger_org ON ledger_entry(org_id);
CREATE INDEX idx_ledger_lease ON ledger_entry(lease_id);
CREATE INDEX idx_ledger_tenant ON ledger_entry(tenant_id);
CREATE INDEX idx_ledger_txn ON ledger_entry(payment_transaction_id);
CREATE INDEX idx_ledger_type ON ledger_entry(type);
CREATE INDEX idx_ledger_date ON ledger_entry(effective_date);
CREATE INDEX idx_ledger_stripe ON ledger_entry(stripe_object_id);
```

## 10. webhook_event

Idempotency store for Stripe webhook events.

**Owning service:** payments-service

```sql
CREATE TABLE webhook_event (
  id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  stripe_event_id   TEXT NOT NULL UNIQUE,
  event_type        TEXT NOT NULL,
  api_version       TEXT,
  stripe_account_id TEXT,
    -- NULL for platform events, acct_xxx for Connect events
  payload           JSONB NOT NULL,
  status            TEXT NOT NULL DEFAULT 'RECEIVED',
    -- RECEIVED, PROCESSING, PROCESSED, FAILED, SKIPPED
  error_message     TEXT,
  retry_count       INTEGER NOT NULL DEFAULT 0,
  processed_at      TIMESTAMPTZ,
  created_at        TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE UNIQUE INDEX idx_webhook_stripe_event ON webhook_event(stripe_event_id);
CREATE INDEX idx_webhook_status ON webhook_event(status) WHERE status IN ('RECEIVED', 'FAILED');
CREATE INDEX idx_webhook_type ON webhook_event(event_type);
```

## Entity-to-Service Ownership Map

- **payments-service** owns all 10 tables above.
- **lease-service** owns `leases` and `tenant_profiles`. Payments-service reads lease data to generate `rent_schedule` and `payment_obligation` records.
- **tenant-service** owns tenant profile data. Payments-service references `tenant_id` by UUID.
- **property-service** owns property and unit data. Payments-service references `property_id` and `unit_id` by UUID.
- **auth-service** owns org and user data. Payments-service references `org_id` by UUID.

## Cross-Service ID References

All cross-service references are **UUID foreign keys by convention** (not enforced at the DB level, since services have separate schemas). Consistency is maintained via:

1. Event-driven sync (EventBridge) for lifecycle changes (lease terminated → cancel future obligations).
2. API calls from payments-service to lease-service/property-service for enrichment at read time.
3. Nightly reconciliation to detect orphaned references.

## Migration Strategy

Migrations will be added to `leasebase-payments-service` as numbered SQL files. The existing seed data structure (from `leasebase-iac/automation/seed/src/seed/payments.ts`) already references `payments`, `payment_methods` in the `payments_service` schema, so the new tables extend that schema.

The existing `payments` table from the scaffold should be migrated to `payment_transaction` and the existing `ledger_entries` table to `ledger_entry` to match the new naming convention.
